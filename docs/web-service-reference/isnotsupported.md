# <a name="isnotsupported"></a>IsNotSupported

Das **IsNotSupported-Element** gibt an, ob die Regel nicht mithilfe der APIs mit verwaltetem Code geändert werden kann. 
  
```XML
<IsNotSupported/>
```

 **Boolescher Wert**
## <a name="attributes-and-elements"></a>Attribute und Elemente

In den folgenden Abschnitten werden Attribute, untergeordnete und übergeordnete Elemente erläutert.
  
### <a name="attributes"></a>Attribute

Keine.
  
### <a name="child-elements"></a>Untergeordnete Elemente

Keine.
  
### <a name="parent-elements"></a>Übergeordnete Elemente

|**Element**|**Beschreibung**|
|:-----|:-----|
|[Regel (RuleType)](rule-ruletype.md) <br/> |Stellt eine Regel im Postfach des Benutzers dar.  <br/> |
   
## <a name="text-value"></a>Textwert

Der Textwert **"true"** gibt an, dass die Regel nicht mithilfe der APIs mit verwaltetem Code geändert werden kann. Der Wert **"false"** gibt an, dass die Regel mithilfe der APIs mit verwaltetem Code geändert werden kann. 
  
## <a name="remarks"></a>HinwBemerkungeneise

Das Schema, das dieses Element beschreibt, befindet sich im virtuellen IIS-Verzeichnis, das Exchange-Webdienste hostet.
